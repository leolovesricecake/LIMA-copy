# 文本版 LIMA（lima_llm）说明

当前分支主流程为 `lima_llm`。  
原始图像版 LIMA 已归档到 `lima_origin/`（见 `lima_origin/说明.md`）。

## 0. 环境准备

### 0.1 Python 环境（暂不使用，继续lima_final）

```bash
conda create -n lima_llm python=3.10 -y
conda activate lima_llm

# 按你的 CUDA 版本选择 PyTorch 安装方式；以下仅示例
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
pip install transformers datasets accelerate sentencepiece tqdm numpy
```

### 0.2 Hugging Face 网络

```bash
# 网络较慢时可用镜像
export HF_ENDPOINT=https://hf-mirror.com
```

## 1. 资源准备（模型与数据）

### 1.1 模型权重

- 默认模型：`Qwen/Qwen2.5-7B-Instruct`
- 远程自动下载：直接把 `--model-path` 设为 HF repo id（如上）
- 本地目录加载：先下载后将 `--model-path` 指向本地目录

示例（已下载到本地）：

```bash
huggingface-cli download Qwen/Qwen2.5-7B-Instruct --local-dir /mnt/huawei/nsq/models/Qwen/Qwen2.5-7B-Instruct
```

### 1.2 数据集（已支持本地 + 远程）

#### SST-2

- 默认远程：`hf://nyu-mll/glue`（config 自动使用 `sst2`）
- 本地：`--sst2-source /path/to/sst2_dir`（目录中有 `validation.tsv/csv/jsonl` 等）
- 远程 URL：`--sst2-source https://.../sst2.zip`（自动下载并缓存）

#### ERASER Movie Reviews

- 默认远程：`hf://eraser-benchmark/movie_rationales`
- 本地：`--eraser-root /path/to/eraser_movie_reviews`
- 远程 URL：`--eraser-root https://.../eraser_movie_reviews.zip`（自动下载并缓存）

说明：
- 远程 URL 下载后会自动缓存和解压（支持 `.zip/.tar.*`）。
- 可用 `--dataset-cache-dir /path/to/cache` 指定缓存目录。

## 2. Dry-run 自检

```bash
python -m lima_llm \
  --dataset sst2 \
  --split validation \
  --mock-backbone \
  --dry-run 10
```

会打印：样本数、标签分布、样本文本长度/Token 数、rationale 覆盖率、chunk 列表。

## 5. 结果文件

- `samples/*.json`: 每样本完整 trace（chunk、每轮选择、四分数、总分）
- `samples/*.txt`: 最终解释文本
- `summary.csv`: 样本级汇总
- `eval_report.json`: Faithfulness/Plausibility/Runtime 报告
- `run_config.json`: 本次运行参数

结果目录命名规则（已包含 `seed`，避免不同随机种子互相覆盖）：

`<output-dir>/<dataset>/model-<model>/chunk-<chunker>_search-<search>_k-<k>_lam-<l1-l2-l3-l4>_seed-<seed>/`


- `gate_b_aggregate.json`: 分组后的均值/方差/通过率
- `gate_b_aggregate.csv`: 便于快速筛选的表格

### 6.3 显存压力调节（新）

若遇到 `CUDA out of memory`，可先降低批量大小（不影响结果，仅影响速度）：

```bash
export LIMA_EMBED_BATCH_SIZE=1
export LIMA_PREDICT_BATCH_SIZE=2
```

说明：
- `LIMA_EMBED_BATCH_SIZE`：控制批量 embedding 的微批大小（默认 `2`）。
- `LIMA_PREDICT_BATCH_SIZE`：控制批量分类概率计算的微批大小（默认 `4`）。

## 7. Gate C（等价性能优化）

Gate C 的原则是：先证明结果等价，再谈提速。当前状态如下：

| 层级 | 任务 | 状态 | 实现位置 | 验收方式 |
| --- | --- | --- | --- | --- |
| P0 | 评估阶段文本概率缓存 | 已实现 | `lima_llm/eval/evaluate.py` 内部 per-sample `prob_cache` | `eval_report.json -> metrics_secondary.eval_prob_cache_stats` |
| P0 | 候选子集打分批处理 | 已实现 | `TextSubmodularObjective.evaluate_gains/evaluate_subsets` + search 调用批接口 | batched/fallback 等价测试 |
| P1 | Objective 缓存键标准化 | 部分完成 | `normalize_subset` 作为 subset key；概率/embedding cache 使用精确文本 key | 待继续统一成显式 key helper |
| P2 | KV cache 复用、候选预筛 | 未实现 | 无 | 后续灰度开关，默认关闭 |

### 7.1 等价性回归

如果环境里有 `pytest`：

```bash
python -m pytest \
  tests/test_lima_llm_gate_c_equivalence.py \
  tests/test_lima_llm_objective_batching.py \
  tests/test_lima_llm_eval_prob_memo.py
```

如果当前环境没有 `pytest`，可以用下面这个无依赖版本：

```bash
python -m compileall lima_llm tests

python - <<'PY'
import importlib.util
import tempfile
from pathlib import Path

def load(path):
    spec = importlib.util.spec_from_file_location(Path(path).stem, path)
    module = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(module)
    return module

gate_c = load("tests/test_lima_llm_gate_c_equivalence.py")
gate_c.test_forward_greedy_batched_and_fallback_paths_are_equivalent()
gate_c.test_bidirectional_batched_and_fallback_paths_are_equivalent()

batching = load("tests/test_lima_llm_objective_batching.py")
batching.test_evaluate_gains_matches_single_evaluate_gain()
batching.test_evaluate_subsets_matches_evaluate_subset()

eval_cache = load("tests/test_lima_llm_eval_prob_memo.py")
eval_cache.test_eval_probability_cache_reduces_duplicate_predict_calls(Path(tempfile.mkdtemp()))

print("Gate C equivalence/cache checks passed")
PY
```

这些检查覆盖：
- batched search 与 fallback search 的 `selected chunks` 一致；
- trace 中的 `marginal_gain / total_score / components` 一致；
- `evaluate_gains/evaluate_subsets` 与逐个 `evaluate_gain/evaluate_subset` 一致；
- eval 阶段概率缓存命中，并且 `misses == predict_calls`。

### 7.3 Gate C 当前边界

- 现在的批处理是“等价批处理”，不会改变搜索目标和 tie-break 规则。
- 环境变量只改变 micro-batch 大小，不应该改变结果。
- KV cache 复用和候选预筛还没有启用；后续必须默认关闭，并配套等价测试后才能进入主流程。
