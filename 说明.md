# 文本版 LIMA（lima_llm）说明

当前分支主流程为 `lima_llm`。  
原始图像版 LIMA 已归档到 `lima_origin/`（见 `lima_origin/说明.md`）。

## 0. 环境准备

### 0.1 Python 环境（暂不使用，继续lima_final）

```bash
conda create -n lima_llm python=3.10 -y
conda activate lima_llm

# 按你的 CUDA 版本选择 PyTorch 安装方式；以下仅示例
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
pip install transformers datasets accelerate sentencepiece tqdm numpy
```

### 0.2 Hugging Face 网络

```bash
# 网络较慢时可用镜像
export HF_ENDPOINT=https://hf-mirror.com
```

## 1. 资源准备（模型与数据）

### 1.1 模型权重

- 默认模型：`Qwen/Qwen2.5-7B-Instruct`
- 远程自动下载：直接把 `--model-path` 设为 HF repo id（如上）
- 本地目录加载：先下载后将 `--model-path` 指向本地目录

示例（已下载到本地）：

```bash
huggingface-cli download Qwen/Qwen2.5-7B-Instruct --local-dir /mnt/huawei/nsq/models/Qwen/Qwen2.5-7B-Instruct
```

### 1.2 数据集（已支持本地 + 远程）

#### SST-2

- 默认远程：`hf://nyu-mll/glue`（config 自动使用 `sst2`）
- 本地：`--sst2-source /path/to/sst2_dir`（目录中有 `validation.tsv/csv/jsonl` 等）
- 远程 URL：`--sst2-source https://.../sst2.zip`（自动下载并缓存）

#### ERASER Movie Reviews

- 默认远程：`hf://eraser-benchmark/movie_rationales`
- 本地：`--eraser-root /path/to/eraser_movie_reviews`
- 远程 URL：`--eraser-root https://.../eraser_movie_reviews.zip`（自动下载并缓存）

说明：
- 远程 URL 下载后会自动缓存和解压（支持 `.zip/.tar.*`）。
- 可用 `--dataset-cache-dir /path/to/cache` 指定缓存目录。

## 2. Dry-run 自检

```bash
python -m lima_llm \
  --dataset sst2 \
  --split validation \
  --mock-backbone \
  --dry-run 10
```

会打印：样本数、标签分布、样本文本长度/Token 数、rationale 覆盖率、chunk 列表。

## 3. 一条命令跑“解释 + 评估”

推荐先用远程 HF 数据集（避免本地数据不完整）：

```bash
bash scripts/run_lima_llm_v1.sh \
  --dataset eraser_movie_reviews \
  --split validation \
  --eraser-root hf://eraser-benchmark/movie_rationales \
  --model-path /mnt/huawei/nsq/models/Qwen/Qwen2.5-7B-Instruct \
  --device cuda:7 \
  --dtype bfloat16 \
  --k 8 \
  --chunker sentence \
  --search greedy \
  --lambdas 1,1,1,1 \
  --output-dir lima_llm_results \
  --dataset-cache-dir /mnt/huawei/nsq/LIMA-copy/datasets \
  --resume-check strict \
  --run-eval --eval-gradient-baseline
```

## 4. 断点恢复

同一配置重复执行，默认会跳过已完成样本：

```bash
python -m lima_llm \
  --dataset eraser_movie_reviews \
  --split validation \
  --eraser-root hf://eraser-benchmark/movie_rationales \
  --model-path /mnt/huawei/nsq/models/Qwen/Qwen2.5-7B-Instruct \
  --device cuda:3 \
  --k 8 \
  --chunker sentence \
  --search greedy \
  --output-dir lima_llm_results \
  --resume-check strict
```

## 5. 结果文件

- `samples/*.json`: 每样本完整 trace（chunk、每轮选择、四分数、总分）
- `samples/*.txt`: 最终解释文本
- `summary.csv`: 样本级汇总
- `eval_report.json`: Faithfulness/Plausibility/Runtime 报告
- `run_config.json`: 本次运行参数

结果目录命名规则（已包含 `seed`，避免不同随机种子互相覆盖）：

`<output-dir>/<dataset>/model-<model>/chunk-<chunker>_search-<search>_k-<k>_lam-<l1-l2-l3-l4>_seed-<seed>/`

## 6. Gate B（多 seed / 小网格）

### 6.1 批量运行

```bash
python scripts/gate_b_run.py \
  --seeds 42,43,44 \
  --k-values 8 \
  --chunkers sentence \
  --searches greedy \
  --lambdas-list '1,1,1,1;1,2,1,1' \
  -- \
  --dataset eraser_movie_reviews \
  --split validation \
  --eraser-root hf://eraser-benchmark/movie_rationales \
  --model-path /mnt/huawei/nsq/models/Qwen/Qwen2.5-7B-Instruct \
  --device cuda:7 \
  --dtype bfloat16 \
  --output-dir lima_llm_results \
  --run-eval --eval-gradient-baseline
```

### 6.2 聚合统计

```bash
python scripts/gate_b_aggregate.py \
  --results-root lima_llm_results \
  --min-runs 3
```

会输出：
- `gate_b_aggregate.json`: 分组后的均值/方差/通过率
- `gate_b_aggregate.csv`: 便于快速筛选的表格
