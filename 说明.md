# 文本版 LIMA（lima_llm）说明

当前分支主流程为 `lima_llm`。
原始图像版 LIMA 已归档到 `lima_origin/`（见 `lima_origin/说明.md`）。

## 1. Dry-run 自检

```bash
python -m lima_llm \
  --dataset sst2 \
  --split validation \
  --mock-backbone \
  --dry-run 10
```

会打印：样本数、标签分布、样本文本长度/Token 数、rationale 覆盖率、chunk 列表。

## 2. 一条命令跑“解释 + 评估”

```bash
bash scripts/run_lima_llm_v1.sh \
  --dataset eraser_movie_reviews \
  --split validation \
  --eraser-root /path/to/eraser \
  --model-path /path/to/Qwen2.5-7B-Instruct \
  --device cuda:0 \
  --dtype bfloat16 \
  --k 8 \
  --chunker sentence \
  --search greedy \
  --lambdas 1,1,1,1 \
  --output-dir lima_llm_results \
  --resume-check strict \
  --run-eval --eval-gradient-baseline
```

## 3. 断点恢复

同一配置重复执行，默认会跳过已完成样本：

```bash
python -m lima_llm \
  --dataset eraser_movie_reviews \
  --split validation \
  --eraser-root /path/to/eraser \
  --model-path /path/to/Qwen2.5-7B-Instruct \
  --device cuda:0 \
  --k 8 \
  --chunker sentence \
  --search greedy \
  --output-dir lima_llm_results \
  --resume-check strict
```

## 4. 结果文件

- `samples/*.json`: 每样本完整 trace（chunk、每轮选择、四分数、总分）
- `samples/*.txt`: 最终解释文本
- `summary.csv`: 样本级汇总
- `eval_report.json`: Faithfulness/Plausibility/Runtime 报告
- `run_config.json`: 本次运行参数
