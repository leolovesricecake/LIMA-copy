## 3. 本方法

统一使用一个脚本：

```
scripts/efficientv2-clip_vitl_multigpu.sh
```

该脚本支持命令行参数配置（`dataset/eval_list/save_dir/cuda_devices/lambda` 等），可覆盖默认值。

### 针对分类正确样本（ImageNet 5k true）

```
bash scripts/efficientv2-clip_vitl_multigpu.sh \
  --dataset /dev/shm/imagenet_val_fast \
  --eval-list datasets/imagenet/val_clip_vitl_5k_true.txt \
  --save-dir submodular_results/imagenet-clip-vitl-efficientv2 \
  --superpixel-algorithm seeds \
  --lambda1 0 --lambda2 0.05 --lambda3 20 --lambda4 1 \
  --pending-samples 8 \
  --cuda-devices "0 1 3 6 7"
```

计算 insertion：

```
python -m evals.eval_AUC_faithfulness --explanation-dir submodular_results/imagenet-clip-vitl-efficientv2/seeds-0.0-0.05-20.0-1.0-pending-samples-8
```

计算最高置信度：

```powershell
python -m evals.evaluation_mistake_debug_ours \
  --explanation-method submodular_results/imagenet-clip-vitl-efficientv2/seeds-0.0-0.05-20.0-1.0-pending-samples-8 \
  --eval-list datasets/imagenet/val_clip_vitl_5k_true.txt
```

注意：

1. `eval_AUC_faithfulness` 对本方法（`{json,npy}/class_id/...` 结构）的进度条是按“类别文件夹数”显示，不是样本数。  
2. 最新脚本会额外打印 `AUC evaluated on xxx samples ...`，以该数字为准。  
3. `wc -l datasets/imagenet/val_clip_vitl_5k_true.txt` 可能显示 `4890`，但该文件实际有 `4891` 条记录（末行无换行符导致口径差异）。

## LIMA 迁移到文本 LLM（v1）

新增代码位于 `lima_llm/`，与原图像版流程解耦。默认是分类/判别任务，支持 `SST-2` 与 `ERASER Movie Reviews`。

### 1. 快速自检（dry-run）

```bash
python -m lima_llm \
  --dataset sst2 \
  --split validation \
  --mock-backbone \
  --dry-run 10
```

输出内容包含：样本数量、标签分布、每条样本字符/Token 长度、rationale 覆盖率，以及首样本 chunk 列表。

### 2. 一条命令完成“解释 + 评估”（推荐）

```bash
bash scripts/run_lima_llm_v1.sh \
  --dataset eraser_movie_reviews \
  --split validation \
  --eraser-root /path/to/eraser \
  --model-path /path/to/Qwen2.5-7B-Instruct \
  --device cuda:0 \
  --dtype bfloat16 \
  --k 8 \
  --chunker sentence \
  --search greedy \
  --lambdas 1,1,1,1 \
  --output-dir lima_llm_results \
  --resume-check strict \
  --run-eval --eval-gradient-baseline
```

### 3. 断点恢复

同一配置重复执行时，默认检查 `json+txt` 产物并跳过已完成样本：

```bash
python -m lima_llm \
  --dataset eraser_movie_reviews \
  --split validation \
  --eraser-root /path/to/eraser \
  --model-path /path/to/Qwen2.5-7B-Instruct \
  --device cuda:0 \
  --k 8 \
  --chunker sentence \
  --search greedy \
  --output-dir lima_llm_results \
  --resume-check strict
```

### 4. 结果目录

运行后会在 `output-dir` 下生成：

- `samples/*.json`：每样本完整 trace（chunk、每轮贪心选择、四分数、总分）
- `samples/*.txt`：最终解释文本（选中 chunk 合并）
- `summary.csv`：样本级汇总
- `eval_report.json`：Faithfulness/Plausibility/Runtime 指标报告
- `run_config.json`：本次运行参数
