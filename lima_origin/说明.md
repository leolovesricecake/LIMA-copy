# 运行位置提示
原始图像版代码已归档到 `lima_origin/`。
请先 `cd lima_origin` 后再执行文档里的历史命令。

代码项目地址：https://github.com/RuoyuChen10/lima

先全部clone下来，正常配置环境pytorch等，安装CLIP（运行代码缺啥补啥吧）。



数据集：下载ImageNet的测试集，放在：datasets/imagenet/ILSVRC2012_img_val



**注：**如果项目不涉及解释错误样本，dataset list为:`datasets/imagenet/val_clip_vitl_5k_true.txt`

涉及错误样本的list为：`datasets/imagenet/val_clip_vitl_2k_false.txt`

数据集默认配置统一在：`dataset_config.py`  
（包含 `dataset-name -> image_root_path / eval_list / save_doc`，可在命令行覆盖）。

## 1. 运行HSIC

打开：`baseline_attribution/generate_explanation_maps_clip_vitl.py`

在 `explainers` 列表中，只保留你要跑的方法（例如仅保留 HSIC）：

``````
explainers = [
        # Saliency(model),
        # GradientInput(model),
        # GuidedBackprop(model),
        # IntegratedGradients(wrapped_model, steps=80, batch_size=batch_size),
        # SmoothGrad(model, nb_samples=80, batch_size=batch_size),
        # SquareGrad(model, nb_samples=80, batch_size=batch_size),
        # VarGrad(model, nb_samples=80, batch_size=batch_size),
        # GradCAM(model),
        # GradCAMPP(model),
        # Occlusion(model, patch_size=10, patch_stride=5, batch_size=batch_size),
        # Rise(model, nb_samples=500, batch_size=batch_size),
        # SobolAttributionMethod(model, batch_size=batch_size),
        HsicAttributionMethod(wrapped_model, batch_size=batch_size),
        # Rise(wrapped_model, nb_samples=500, batch_size=batch_size),
        # Lime(model, nb_samples = 1000),
        # KernelShap(wrapped_model, nb_samples = 1000, batch_size=32)
    ]
``````

然后：

```
python -m baseline_attribution.generate_explanation_maps_clip_vitl \
  --dataset-name imagenet \
  --device 1
```

结果请见：`explanation_results`文件夹

然后：

```
python -m baseline_attribution.debug_org_attribution_method_clip_vitl \
  --dataset-name imagenet \
  --explain-method HsicAttributionMethod \
  --device 1
```

得到结果在`explanation_insertion_results`

## 2. 运行GradECLIP

打开：`baseline_attribution/generate_explanation_maps_Grad-ECLIP.py`

按脚本中注释切换 Grad-ECLIP 对应配置（如果项目涉及筛选预测错误样本子集，请保留相关逻辑）。

然后：

```
python -m baseline_attribution.generate_explanation_maps_Grad-ECLIP \
  --dataset-name imagenet \
  --device 1
```

结果请见：`explanation_results`文件夹

然后：

```
python -m baseline_attribution.debug_org_attribution_method_clip_vitl \
  --dataset-name imagenet \
```

## 3. 本方法

统一使用一个脚本：

```
scripts/efficientv2-clip_vitl_multigpu.sh
```

该脚本支持命令行参数配置（`dataset/eval_list/save_dir/cuda_devices/lambda` 等），可覆盖默认值。

### 针对分类正确样本（ImageNet 5k true）

```
bash scripts/efficientv2-clip_vitl_multigpu.sh \
  --dataset /dev/shm/imagenet_val_fast \
  --eval-list datasets/imagenet/val_clip_vitl_5k_true.txt \
  --save-dir submodular_results/imagenet-clip-vitl-efficientv2 \
  --superpixel-algorithm seeds \
  --lambda1 0 --lambda2 0.05 --lambda3 20 --lambda4 1 \
  --pending-samples 8 \
  --cuda-devices "0 1 3 6 7"
```

计算 insertion：

```
python -m evals.eval_AUC_faithfulness --explanation-dir submodular_results/imagenet-clip-vitl-efficientv2/seeds-0.0-0.05-20.0-1.0-pending-samples-8
```

计算最高置信度：

```powershell
python -m evals.evaluation_mistake_debug_ours \
  --explanation-method submodular_results/imagenet-clip-vitl-efficientv2/seeds-0.0-0.05-20.0-1.0-pending-samples-8 \
  --eval-list datasets/imagenet/val_clip_vitl_5k_true.txt
```

注意：

1. `eval_AUC_faithfulness` 对本方法（`{json,npy}/class_id/...` 结构）已支持递归子目录（例如 `class_id/n014.../*.json|npy`）。  
2. 最新脚本会额外打印 `AUC evaluated on xxx samples ...`，以该数字为准。  
3. `wc -l datasets/imagenet/val_clip_vitl_5k_true.txt` 可能显示 `4890`，但该文件实际有 `4891` 条记录（末行无换行符导致口径差异）。
